Cache-Control: proxy-revalidate

CacheControlProxyRevalidate.html
Cache Type Will Cache?
Browser Yes, Ignore
Proxy Yes

This header is part of the HTTP/1.1 standard and instructs proxy caches to verify the status of the stale resources before using it and expired ones should not be used, i.e. The proxy should check if the resource is fresh everytime the resource is needed, even if the expiration date has not been reached. This directive is ignored by private caches 


                    # Webserver config
                    server {
                        # ...
                        location /CacheControlProxyRevalidate.html {
                            add_header Cache-Control "proxy-revalidate";
                            add_header "Expires" @1h; # Set expires to 1hr from now
                        }
                    }
