This page was sent with the Surrogate-Control response header allows origin servers to dictate how surrogates (i.e. gateway caches) should handle response entities, with control directives. It is part of a draft spec, but some popular gateway caches (i.e. Squid and Fastly) already support it. The gateway caches that don't support Surrogate-Control usually have their own custom header that works the same way (i.e. Akamai has Edge-Control).

Currently defined directives control processing and cache behavior. 


                      # Webserver config
                      server {
                          # ...
                          location /SurrogateControl.html {
                              add_header Surrogate-Control "max-age=3600";
                      }
