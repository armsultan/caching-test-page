Cache-Control: must-revalidate

CacheControlMustRevalidate.html
Cache Type Will Cache?
Browser Yes
Proxy Yes

This header is part of the HTTP/1.1 standard and instructs the cache to verify the status of the stale resources before using it and expired ones should not be used, i.e. The client should check if the resource is fresh everytime the resource is needed, even if the expiration date has not been reached.

This might still be a performance boost in most cases because if the resource has not been modified, the server will return a 304 (not modified), which is likely more lightweight compared to your original resource. 


                    # Webserver config
                    server {
                        # ...
                        location /CacheControlMustRevalidate.html {
                            add_header Cache-Control "must-revalidate";
                            add_header "Expires" @1h; # Set expires to 1hr from now
                        }
                    }
