This page was sent with the Expires response header with a future date, for example, Expires: Thu, 31 Dec 2037 23:55:55 GMT. This header is part of the HTTP/1.0 standard, and deprecated in HTTP 1.1 and should be avoid when possible, but it is still widely supported and used. If there is a Cache-Control header with the max-age or "s-maxage" directive in the response, the Expires header is ignored.

Invalid dates, like the value 0, represent a date in the past and mean that the resource is already expired. 


                  # Webserver config
                  server {
                      # ...
                      location /Expires.html {
                        add_header "Expires" @1h; # Set expires to 1hr from now
                      }
                  }
